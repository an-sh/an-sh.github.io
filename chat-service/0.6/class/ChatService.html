Class: ChatService

Defined in: lib/chat.coffee

Overview

An instance creates a new chat service.

Instance Method Summary

  • # (void) close(done = function() {}) Closes server.

Constructor Details

# (void) constructor(options = {}, hooks = {}, state = 'memory')

Server creation/integration.

Parameters:

  • options ( Object ) — Options.
  • hooks ( Object ) — Hooks. Every UserCommand is wrapped with the corresponding Before and After hooks. So a hook name will look like roomCreateBefore. Before hook is ran after arguments validation, but before an actual server command processing. After hook is executed after a server has finshed command processing. Check Hooks unit tests section in the test directory for more details and hooks usage examples.
  • state ( String or Constructor ) — Chat state. Can be either 'memory' or 'redis' for built-in state storage, or a custom state constructor function that implements the same API.

Options Hash: (options):

  • namespace ( String ) — io namespace, default is '/chat-service'.
  • historyMaxMessages ( Integer ) — room history size, default is 100.
  • useRawErrorObjects ( Boolean ) — Send error objects (see ErrorBuilder) instead of strings, default is false.
  • enableUserlistUpdates ( Boolean ) — Enables ServerMessages#roomUserJoined and ServerMessages#roomUserLeft messages, default is false.
  • enableAdminlistUpdates ( Boolean ) — Enables ServerMessages#roomAdminAdded and ServerMessages#roomAdminRemoved messages, default is false.
  • enableDirectMessages ( Boolean ) — Enables user to user UserCommands#directMessage, default is false.
  • serverOptions ( Boolean ) — Options that are passed to socket.io if server creation is required.
  • stateOptions ( Boolean ) — Options that are passed to a service state.
  • io ( Object ) — Socket.io instance that should be used by ChatService.
  • http ( Object ) — Use socket.io http server integration.

Options Hash: (hooks):

  • auth ( Function ) — Socket.io auth hook. Look in the socket.io documentation.
  • onConnect ( Function(<ChatService>, <Socket>, <Function(<Error>, <String>, <Object>, <Object>)>) ) — Client connection hook. Must call a callback with either error or user name, auth data and user state. User name and auth data are send back with loginConfirmed message. Error is sent as loginRejected message. User state is the same as User#initState.
  • onClose ( Function(<ChatService>, <Error>, <Function(<Error>)>) ) — Executes when server is closed. Must call a callback.
  • onStart ( Function(<ChatService>, <Function(<Error>)>) ) — Executes when server is started. Must call a callback.
