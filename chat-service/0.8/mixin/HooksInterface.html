Mixin: HooksInterface

Defined in: src/ChatService.coffee

Overview

Hooks interface. Hooks can either return a Promise or call a callback. The ChatService#constructor hooks parameter expects an object that contains any of the described hooks.

Method Summary

  • ~ (Promise<Array>) onConnect(server, id, cb) Client connection hook.
  • ~ (Promise) onStart(server, cb) Executes when server is started (after a state and a transport are up, but before message processing is started).
  • ~ (Promise) onClose(server, cb) Executes when server is closed (after a transport is closed and all clients are disconnected, but a state is still up).
  • ~ (Promise) directMessagesChecker(msg, cb) Validator for UserCommands#directMessage message objects.
  • ~ (Promise) roomMessagesChecker(msg, cb) Validator for UserCommands#roomMessage message objects.
  • ~ (Promise<null> or Promise<Array>) _COMMAND_Before(execInfo, cb) Before hooks are available for all UserCommands and are executed after an arguments validation.
  • ~ (Promise<null> or Promise<Array>) _COMMAND_After(execInfo, cb) After hooks are available for all UserCommands and are executed after ChatService default event handlers.

~ (Promise<Array>) onConnect(server, id, cb)

Client connection hook.

Parameters:

  • server ( ChatService ) — Service instance.
  • id ( String ) — Socket id.
  • cb ( Callback ) — Optional callback.

Returns:

  • ( Promise<Array> ) — Returns an array with a login string (user name) and an optional auth data object. User name and auth data are send back with a ServerMessages#loginConfirmed message. Error is sent as a ServerMessages#loginRejected message.

~ (Promise) directMessagesChecker(msg, cb)

Validator for UserCommands#directMessage message objects. When is set, a custom format in direct messages is enabled. When hooks resolves, than a message format is considered valid, and the other way around for the rejection case.

~ (Promise<null> or Promise<Array>) _COMMAND_Before(execInfo, cb)

Note: Substitute _COMMAND_ with one of UserCommands.

Before hooks are available for all UserCommands and are executed after an arguments validation.

Parameters:

  • execInfo ( ExecInfo )
  • cb ( Callback ) — Optional callback.

Returns:

  • ( Promise<null> or Promise<Array> ) — Resolves without a data to continue a command execution. Rejections or a resolved array will stop further execution, and return results to the command issuer.

~ (Promise<null> or Promise<Array>) _COMMAND_After(execInfo, cb)

Note: Substitute _COMMAND_ with one of UserCommands.

After hooks are available for all UserCommands and are executed after ChatService default event handlers.

Parameters:

  • execInfo ( ExecInfo )
  • cb ( Callback ) — Optional callback.

Returns:

  • ( Promise<null> or Promise<Array> ) — Resolves without a data to return unchanged command results to the command issuer. Rejections or a resolved array will override command results.
